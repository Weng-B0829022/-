HETA 候選邊 R 值分布：兩種極端情況分析
==========================================


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
情況一：SD >= Mean（T_I 為負）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

條件：
  候選邊 R 值分布的標準差 >= 平均值

數學推導：
  T_I = Mean - SD <= 0

判斷條件（論文原文為嚴格大於）：
  R > T_I
  → 任何非負的 R 值（R >= 0）都滿足 R > T_I（負數）
  → 所有候選邊均被判定為 local bridge

結果：
  → global bridge 消失
  → 沒有任何候選邊的 R 值低於 T_I 而被留到下一層
  → 所有邊在某一層被分類完畢

關鍵結論：
  SD >= Mean  →  T_I <= 0  →  global bridge 消失


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
情況二：候選邊全為 0（T_I 恰好等於 0）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

條件：
  所有候選邊的 R 值均等於 0

數學推導：
  Mean = 0，SD = 0
  T_I = 0 - 0 = 0

判斷條件（論文原文為嚴格大於）：
  R > T_I
  → 0 > 0  →  False
  → 所有候選邊均不滿足條件，無法被判定為 local bridge
  → 全部流向下一層，若各層情況相同，最終抵達 Step 3.3

結果：
  → global bridge 全部出現
  → 所有候選邊最終被判定為 global bridge

關鍵結論：
  候選邊全為 0  →  T_I = 0  →  嚴格大於條件不成立  →  global bridge 全部出現

兩種情況對比
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  情境             T_I    判斷結果        global bridge
  ──────────────  ─────  ──────────────  ─────────────
  SD > Mean        < 0   R > 負數 → 成立   消失
  候選邊全為 0     = 0   0 > 0    → 不成立  全部出現

補充說明：
  兩者同為 SD >= Mean 的子集，卻因 T_I 是否嚴格小於 0
  導致 global bridge 的分類結果完全相反。
  此邊界行為對使用 >= 的實作（如 heta.py）會產生 bug：
  當 T_I = 0 且 R = 0 時，>= 條件成立，邊被誤判為 local bridge，
  而非論文定義的 global bridge。
